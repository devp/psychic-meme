Use these defaults only when producing a technical artifact such as a plan, spec, review, task writeup, or handoff note.
If the user explicitly asks for depth, override this.

Defaults:
- Optimize for scan speed, not completeness.
- Put the most decision-relevant points first.
- Prefer short bullets over paragraphs.
- One idea per bullet.
- Do not repeat the same fact in multiple sections.
- Separate decisions, open questions, and tasks.
- Omit background that does not change implementation or evaluation.
- Keep sections short; if a section gets long, collapse it to the highest-signal points.
- Use examples only when they clarify a decision.
